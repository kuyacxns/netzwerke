IP-Adresse
Internet Protocol Address
Wie die Postadresse deines Hauses – eindeutig, damit Datenpakete ankommen.

Jedes Gerät in einem Netzwerk braucht eine eindeutige Adresse – die IP-Adresse. Ohne sie wüsste das Netzwerk nicht, wohin ein Datenpaket soll.

Aufbau einer IPv4-Adresse
192.168.10.5
192 = Grobe Region 168 = Netzbereich 10 = Subnetz 5 = Gerät
Private IP (im eigenen Netz)
192.168.x.x

Nur innerhalb des lokalen Netzwerks gültig. Kein Gerät im Internet hat diese Adresse – sie sind reserviert für private Netze.

Öffentliche IP (im Internet)
85.214.132.117

Weltweit eindeutig. Diese Adresse bekommt dein Router vom Internetanbieter – über sie bist du im Internet erreichbar.

LAN & WAN
Local Area Network / Wide Area Network
LAN = dein Zuhause. WAN = die öffentliche Strasse. Der Router steht in der Haustür dazwischen.
LAN – Lokales Netzwerk

Alle Geräte an einem Standort – Wohnung, Büro, Schulgebäude. Verbunden per Kabel (Ethernet) oder WLAN.

Reichweite: innerhalb eines Gebäudes Geschwindigkeit: bis 10 Gbit/s Beispiel: unser Schulnetzwerk
WAN – Weitverkehrsnetz

Verbindet mehrere LANs über große Entfernungen. Das Internet ist das größte WAN der Welt.

Reichweite: städte- bis weltweit Verbindung: Glasfaser, DSL, Satellit Beispiel: das Internet

Der Router übersetzt zwischen der privaten LAN-Adresse und der öffentlichen WAN-Adresse (NAT).

VLAN
Virtual Local Area Network
Stockwerke im Bürogebäude: alle nutzen dasselbe Gebäude (Switch), aber jeder Stock (VLAN) ist ein eigener Bereich.

Ein VLAN teilt ein physisches Netzwerk in logisch getrennte Bereiche auf. Geräte in verschiedenen VLANs können nicht direkt miteinander kommunizieren – nur über einen Router.

VLANNameNetzGeräte
10Management192.168.10.0/24Router, Raspberry Pi, Admins
20Clients192.168.20.0/24Laptops, PCs, Smartphones
30Server192.168.30.0/24Webserver, DNS, Wiki, LDAP
Warum VLANs?
1
Sicherheit: Ein Angreifer im Client-VLAN kommt nicht an die Server heran.
2
Ordnung: Jeder Bereich hat seinen eigenen IP-Adressbereich – übersichtlich.
3
Performance: Broadcast-Pakete bleiben im eigenen VLAN – kein Lärm im ganzen Netz.
Benötigt: Managed Switch

Ein einfacher (unmanaged) Switch versteht keine VLANs. Wir brauchen einen Managed Switch, der VLAN-Tags (802.1Q) versteht und den Datenverkehr entsprechend trennt.

Router
Netzwerk-Wegewahl-Gerät
Postverteilzentrum: Pakete kommen an und werden je nach Adresse in die richtige Richtung weitergeschickt.

Ein Router verbindet zwei oder mehr Netzwerke und leitet Datenpakete zwischen ihnen weiter. Er entscheidet anhand der Ziel-IP, wohin ein Paket geschickt wird.

Aufgaben des Routers in unserem Projekt
→ LAN mit dem Internet (WAN) verbinden
→ Datenverkehr zwischen den 3 VLANs routen
→ NAT durchführen (private ↔ öffentliche IP)
→ Standard-Gateway für alle Geräte

OpenWRT auf dem Raspberry Pi

In unserem Projekt läuft der Router als Software (OpenWRT) auf einem Raspberry Pi. OpenWRT ist ein Linux-basiertes Router-Betriebssystem mit vollem VLAN-, Firewall- und VPN-Support.

Switch
Netzwerk-Verteiler
Mehrfachsteckdose: viele Geräte anschließen, Strom (Datenpakete) fließt gezielt nur dorthin, wo er gebraucht wird.

Ein Switch verbindet mehrere Geräte innerhalb desselben Netzwerks. Im Gegensatz zum Hub schickt er Pakete nur an den Empfänger – nicht an alle.

Hub (veraltet)

Schickt jedes Paket an alle angeschlossenen Geräte. Ineffizient, unsicher. Kaum noch im Einsatz.

Switch (modern)

Schickt Pakete gezielt an den richtigen Empfänger. Merkt sich MAC-Adressen in einer Tabelle.

DHCP
Dynamic Host Configuration Protocol
Empfangsdame im Hotel: du checkst ein, sie gibt dir automatisch einen Zimmerschlüssel – du musst dir keine Nummer selbst aussuchen.

DHCP sorgt dafür, dass Geräte beim Eintritt ins Netzwerk automatisch eine IP-Adresse und alle weiteren Netzwerkeinstellungen bekommen – ohne manuelle Konfiguration.

Was DHCP alles verteilt
IP-Adresse z.B. 192.168.20.105
Subnetzmaske 255.255.255.0
Standard-Gateway 192.168.20.1 (Router)
DNS-Server 192.168.20.2 (Pi)
DHCP-Ablauf (DORA)
DDiscover: Gerät ruft "Gibt es hier einen DHCP-Server?"
OOffer: Server antwortet "Ich biete dir 192.168.20.105 an"
RRequest: Gerät sagt "Ja, ich nehme 192.168.20.105"
AAcknowledge: Server bestätigt "OK, sie gehört dir für 24h"
Konfiguration in unserem Projekt (dnsmasq)
dhcp-range=192.168.20.100,192.168.20.200,24h
dhcp-option=3,192.168.20.1
dhcp-option=6,192.168.20.2

Webserver
HTTP-Server (Apache / Nginx)
Bibliothekarin: du fragst nach einem Buch (URL), sie reicht es dir (HTML-Seite).

Ein Webserver empfängt HTTP/HTTPS-Anfragen von Browsern und liefert Webseiten, Bilder und Daten zurück. Er "hört" standardmäßig auf Port 80 (HTTP) und 443 (HTTPS).

Wiki-Server
Internes Wissensmanagementsystem
Firmeninternes Wikipedia: Wissen für alle Teammitglieder zugänglich und gemeinsam bearbeitbar.

Ein Wiki-Server ist eine Webanwendung, mit der ein Team gemeinsam Dokumentationen schreiben und verwalten kann. In unserem Projekt ideal für Konfigurationsanleitungen und Troubleshooting-Notizen.

Beliebte Wiki-Software

LDAP
Lightweight Directory Access Protocol
Zentraler Hausschlüsseldienst: ein einziger Schlüssel öffnet alle Türen im Gebäude – ein Login für alle Dienste.

LDAP ist ein Protokoll zur zentralen Benutzerverwaltung. Alle Dienste (Wiki, Webserver, etc.) fragen den LDAP-Server, ob ein Benutzer sich einloggen darf.

Mit und ohne LDAP
Ohne LDAP
Jeder Dienst hat eigene Benutzerdatenbank. Anderes Passwort für Wiki, Webserver, Server-Login... unübersichtlich.
Mit LDAP
Ein zentrales Passwort für alle Dienste. Benutzer einmal anlegen – überall nutzbar. Zugriffsrechte zentral steuern.
Software: OpenLDAP auf dem Raspberry Pi

OpenLDAP ist die verbreiteste Open-Source-Implementierung von LDAP. Installation: sudo apt install slapd ldap-utils

Open-Source Router-Betriebssystem

OpenWRT ist ein Linux-basiertes Betriebssystem für Router. Es ersetzt die eingeschränkte Standard-Firmware und gibt dir vollständige Kontrolle über dein Netzwerk.

Firewall
Netzwerk-Schutzwall
Sicherheitsmann am Eingang mit einer Liste: wer drauf steht, kommt rein – alle anderen werden gestoppt.

Eine Firewall überwacht und filtert den Netzwerkverkehr nach vordefinierten Regeln. Sie entscheidet: erlaubt oder blockiert?

Beispiel-Regeln für unser Projekt (iptables)
SSH (Port 22) aus VLAN 10 → ERLAUBT
HTTP (Port 80) von überall → ERLAUBT
HTTPS (Port 443) von überall → ERLAUBT
SSH aus VLAN 20 oder 30 → BLOCKIERT
Direkte VLAN20↔VLAN30-Kommunikation → BLOCKIERT
Alles andere → DROP
Software vs. Hardware Firewall
Software-Firewall
Läuft als Programm auf dem Raspberry Pi (iptables/nftables). Günstig, flexibel.
Hardware-Firewall
Eigenes Gerät zwischen Router und Netz (z.B. pfSense). Leistungsfähiger für große Netze.
NAT
Network Address Translation
Eine Firmenzentrale mit einer einzigen Postadresse nach außen – intern gibt es viele Abteilungen mit eigenen Nummern.

NAT übersetzt private IP-Adressen (intern) in eine öffentliche IP-Adresse (extern) – und zurück. So können viele Geräte mit einer einzigen öffentlichen IP ins Internet.

NAT in Aktion

→

NAT-Übersetzung
→
Internet
85.214.x.x

Der Router merkt sich: "Paket von 192.168.10.5 geht nach draußen als 85.214.x.x". Kommt die Antwort zurück, leitet er sie an .5 weiter.

Warum brauchen wir NAT?

Es gibt weltweit nur ca. 4,3 Milliarden IPv4-Adressen – zu wenig für alle Geräte. Private IP-Ranges (192.168.x.x, 10.x.x.x) werden millionenfach wiederverwendet. NAT macht das möglich, weil nach außen nur eine öffentliche IP sichtbar ist.

VPN
Virtual Private Network
Ein sicherer, privater Tunnel durch die öffentliche Strasse: von außen sieht man den Tunnel, aber nicht was darin transportiert wird.

Ein VPN erstellt eine verschlüsselte Verbindung durch ein öffentliches Netzwerk. So kann man sicher auf ein privates Netzwerk zugreifen – auch von unterwegs.

DMZ
Demilitarisierte Zone
Eingangsbereich (Foyer): Besucher kommen ins Foyer, aber nicht tiefer ins Gebäude. Ein Einbruch ins Foyer gefährdet nicht den Rest.

Die DMZ ist ein separiertes Netzwerksegment zwischen dem Internet und dem internen Netz. Öffentlich erreichbare Dienste (z.B. Webserver) stehen hier.

Netzwerkstruktur mit DMZ
Internet
→ darf in die DMZ
Firewall 1
DMZ (Webserver)
← Internet-Zugriff erlaubt
Firewall 2
Internes LAN
← kein direkter Internet-Zugriff
Vorteil der DMZ

Wird der Webserver in der DMZ kompromittiert (gehackt), hat der Angreifer keinen direkten Zugang zum internen Netz mit sensiblen Daten. Die zweite Firewall schützt das interne LAN.

Subnetzrechner
IPv4-Subnet Calculator – Python-Tool
Taschenrechner für Netzwerke: du gibst IP und Maske ein, er berechnet alles automatisch.

Wir programmieren ein Python-Tool, das aus IP-Adresse und Subnetzmaske alle wichtigen Netzwerkinfos berechnet. Jedes Teammitglied muss den Code erklären können!

Interaktiver Rechner
IP-Adresse
CIDR
Python-Struktur (Pflichtanforderungen)
import ipaddress

def berechne_subnetz(ip_eingabe, maske):
    # Netzwerk-Objekt erstellen
    netz = ipaddress.IPv4Network(
        f'{ip_eingabe}/{maske}', strict=False
    )
    # Werte berechnen
    netz_id   = netz.network_address
    broadcast = netz.broadcast_address
    hosts     = netz.num_addresses - 2
    maske_str = netz.netmask
    return netz_id, broadcast, hosts, maske_str

OSI-Modell
Open Systems Interconnection – 7-Schichten-Modell
Wie das Verschicken eines Briefes: Du schreibst den Inhalt (Schicht 7), packst ihn in einen Umschlag und adressierst ihn (Schichten 6–3), und schliesslich trägt ihn der Briefträger auf der physischen Strasse aus (Schicht 1).

Das OSI-Modell unterteilt die gesamte Netzwerkkommunikation in 7 Schichten. Jede Schicht hat eine klar definierte Aufgabe und nutzt nur die Dienste der Schicht direkt darunter. So lässt sich gezielt benennen, auf welcher Ebene ein Problem liegt.

6

Datenformatierung, Verschlüsselung, Komprimierung
TLS/SSL · JPEG · ASCII
5
Sitzung Session
Verbindungsauf- und -abbau, Sitzungsverwaltung
NetBIOS · RPC · SSH
Transportschicht (4)
4
Transport Transport
Zuverlässige Übertragung, Ports, Fehlerkontrolle, Segmentierung
TCP · UDP · Port 80/443/22
Infrastrukturschichten (3–1)
3

Warum brauchen wir das OSI-Modell?
1
Fehlersuche: "Das Problem liegt auf Schicht 3" – sofort klar: IP-Routing-Problem, kein Kabeldefekt.
2
Interoperabilität: Hersteller bauen Geräte, die auf bestimmten Schichten zusammenarbeiten – Switch (Schicht 2), Router (Schicht 3).
3
Gemeinsame Sprache: Netzwerker weltweit sprechen dieselbe Schichtensprache – "Layer 2", "Layer 7 Protokoll".

Datenkapselung: So wird ein HTTP-Paket "eingepackt"

Beim Senden fügt jede Schicht einen Header hinzu (Enkapsulation). Beim Empfänger wird er Schicht für Schicht wieder entfernt (Dekapsulation).

[Schicht 7]
HTTP-Daten: "GET /index.html HTTP/1.1"
[Schicht 4]
TCP-Header HTTP-Daten
[Schicht 3]
IP-Header TCP-Header | HTTP-Daten
[Schicht 2]
Ethernet-Frame IP | TCP | HTTP-Daten
[Schicht 1]
⚡ 0101101010111001... (Bits über das Kabel)

Beim Empfänger (z.B. Webserver auf 192.168.30.5) wird jede Schicht der Reihe nach wieder "ausgepackt".
